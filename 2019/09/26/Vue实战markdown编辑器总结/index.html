Vue实战markdown编辑器总结

项目实战第一实战: 创建一个markdown的小型编辑器

涉及的知识点: 1.Markdown转换库 2.侦听器watcher ** ** 3.生命周期函数 ****

4.本地缓存API:localStorage.setItem 与 localStorage.getItem.5. 过滤器filter

涉及的技巧和方法 : 1.进行合理的规划. 2.初始化数据的基本两种方法. 3.利用v-if防止页面出错. 4.利用JSON字符串的转换实现localStorageAPI的成功调用.

合理的规划

  • 对于一个已知项目,应该从最核心的功能开始实现,首先对于一个markdown编辑器,最重要的便是要有写的地方,
  • 并且能通过markdown转换成html语法展示在页面上.
  • 通过v-model 和 v-html同步实现,需要markdown语法转换CDN,通过script引入.
  • 确定该基本的功能后,考虑其他功能的实现,比如多笔记页面的展示(左边有选择笔记的sidebar)
  • 删除,收藏,添加等功能(toolbar)的实现.
  • 文字字数行数等辅助功能的实现.
  • 在做的过程中,利用CSS来布局各个功能部件.

本地缓存localStorage.setItem()于localStorage.getItem()

  • 浏览器提供两个API，可以将本次使用的对象的值到浏览器本地的Storage中，并且在下次刷新浏览器时可以再次调用.
  • 需要注意的是,这种方法不要经常用,会造成浏览器的负荷增大,我们一般的用法都是通过后端传输数据。
  • 具体用法,localStorage.setItem(valName,value)接受两参数,valName是浏览器本地缓存对象的名字,value是本地缓存对象的值。
  • localStorage.getItem(valName)接受一个参数，valName是setItem中设定的本地缓存对象的名字,返回value。下面将掩饰具体使用.

JSON字符串的转换实现localStorageAPI

  • 这里是一个细节，如果我们要进行本地存储的是一个对象数组，并且直接进行setItem的话，会造成错误，因为localStorage只接受字符串.
  • 利用JSON.stringify(notes)，将需要存储的对象数组转化为json字符串存储.
  • 那么对应的从中取出来的也是json字符串，便再使用json字符串解析方法 JSON.parse()即可.
