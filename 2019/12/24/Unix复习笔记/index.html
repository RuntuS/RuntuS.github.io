Unix复习笔记

1.Unix操作系统概述

Unix功能特征

  1. 交互式$\color{red}{分时多用户}$（这是最重要的)
  2. 软件复用
  3. 可移植性强
  4. 配置灵活，适应性强
  5. 界面方便高效
  6. 安全机制完善
  7. 多国语言支持
  8. 支持多种网络协议

2.Unix操作系统入门

帐号控制

  • 登陆: login
  • 退出: exit

  • 修改密码: passwd

  • 用户基本信息文件在 /etc/passwd

简单命令

  1. date : 显示当前日期.

  2. who : 查看所有已登陆用户情况.

  • $\color{red}{注:一个用户名可以被一个人多次登陆或者被多个人同时登陆，即不会像玩游戏一样被挤下线，who命令查看时可能存在多个相同名字的用户在线。}$
  1. 通用命令格式
  • $ 命令名 [-命令行任选项] [命令参数]
  • 注意方括号表示其中的内容可以缺省。
  • 命令行任选项通常由多个字符组成，命令参数主要用于指明操作对象，一般来说都是文件名。
  1. 命令行输入

  • ^h删除前面字符。 ^u删除一行字符。

  • 联机手册 man 命令

  • 注意#是超级用户提示符。(root权限)

3.编辑程序vi

编辑器类型

  • 行编辑器 - 编辑单位为行 – 命令为edit,e,ex
  • 屏幕编辑器 – 编辑单位为字符 – 命令为vi,v,view
  • 而这里介绍的vi编辑器则是屏幕编辑器。

vi编辑器

  • vi两种模式
  1. 命令模式: 进入vi的初始模式，例如移动光标，粘贴，查找文本等。
  2. 文本输入模式: 命令模式的子模式，除了ESC键外的所有键都作为键入字符添加到文本中，按ESC键返回命令模式，才可进行操作。

  • vi的基本使用

  • 进入vi:

  • 开始编辑文件filename，如果filename不存在，则新创建一个文件。

  • 退出vi(前提:在命令模式下)

  • 发出:wq命令，保存文件内容并且退出。（write quit)

  • 从命令模式到文本模式命令键

  1. i -> 光标前插入文本
  2. I -> 光标所在行首插入文本
  3. a -> 光标后加入文本
  4. A -> 光标所在行尾加入文本
  5. o -> 光标所在行下新加一行
  6. O -> 光标所在行上新加一行

  • 命令模式键位

  • h -> 左 ；j -> 下 ； k -> 上 ； l -> 右

  • x -> 删除光标所占字符 ， dw -> 删除从光标到单词结束所有字符.

  • D -> 删除光标所在位置到行尾部分

  • dd -> 删除光标所在行

  • 保存文本，退出编辑程序

  • :wq 保存文本，退出

  • :w 保存文本，不退出

  • :w file 文本保存到file里，不退出

  • :q 退出vi(如果不保存，则会报错，然后被拒绝退出)

  • :q! 强制退出

4. Unix文件系统

Unix文件类型

  • 注意:对于Unix操作系统来说，文件是字节序列。
  1. 普通文件，用vi编辑器创建的文件都是普通文件。
  2. 目录文件，注意文件夹（即目录文件）也是文件。
  3. 特殊文件，包含与外部系统，如打印机等相联系的特定信息。

  • 重要知识点: $\color{red}{文件和进程是Unix系统中的两个最基本的概念。}$

  • 路径名

  1. 相对路径: 起始点是当前路径。即开头是以当前路径名开头，例如Unix/fuxi.md
  2. 绝对路径: 起始点是根路径。例如/Desktop/gooda
  • 文件名
  • 所有的Unix系统至少允许14个字符长的文件名，大多数系统支持255个字符的文件名。
  • Unix系统对大小写敏感，大写和小写是不同的意思。
  • 文件名可用字符。
  1. ( A~Z ) 大写字母
  2. ( _ ) 下划线
  3. ( a~z ) 小写字母
  4. ( . ) 小数点
  5. ( 0~9 ) 数字
  • 注意: 以(.)开头的文件是隐含(隐藏)文件名，在列目录操作时候通常不显示出来。

目录操作

1. pwd ： 显示当前工作路径。
2. cd 改变当前工作路径。
注: cd后面可以接绝对路径，也可以接相对路径，有几个特殊的用法下面会说明。

补充一下 : *主目录:** 系统管理员在系统上创建所有的帐号，并且为每个帐号都分配一个特定的目录，这个目录就是主目录。

3. mkdir -> 创建目录，且可以同时创新多个目录。（用户在创建目录的父目录具有写权限)

4. rmdir -> 删除目录, 删除的目录是空目录。(用户在要被删除目录的父目录下具有写权限。)

5. ls [-options] [filenames]

  • filenames如果是普通文件，则列出文件基本信息，如果是目录文件，则列出目录下的文件信息.
  • ls命令常用任选项

ls -l 重要命令. 展示文件七项主要信息。

文件名匹配

  • 通配符
  1. ? -> 匹配任何单个字符
  2. * -> 匹配任何多个字符
  3. [list] -> 匹配list中的任意单个字符
  4. [!list] -> 匹配不在list中的任意单个字符

文件管理命令

  1. cat命令 显示文件内容（一次性全部显示）。
  2. more命令 显示文件内容 (逐屏显示)。
  3. rm命令 删除文件
  • rm -i 删除前确认是否删除。 rm -r 强制删除文件，可用于删除非空目录文件（注意和rmdir区别)
  1. cp命令 复制文件
  • cp [-i][-r] oldfile newfile
  • 将oldfile的内容拷贝到newfile中，newfile如果已存在，则覆盖。
  • -i : 如果newfile已存在，则会请求一次。
  • -r : 将已存在的目录下的所有内容拷贝到新的目录下。
  • 如果newfile是目录文件，则会将oldfile原封不动的拷贝到newfile目录下，且名字相同。
  1. mv命令 移动文件
  • mv aaa bbb
  • 将aaa文件更名为bbb或者将aaa文件移动到bbb目录下，并且源文件不存在（注意和cp指令区别)
  • 细节情况与cp相同，这里不加起叙述。
  1. ln命令 链接文件
  • 相当于给文件再取一个名字。这里把名字当作指针，相当于再创建一个指针指向内容，操作任何一个指针（名字)，都会改变相应的文件的内容，如果删除其中一个文件名，只要名字没有删除完，则文件就不会消失。

基本使用程序(这个很重要，特别是在shell编程中突出明显)

  1. 文件统计wc命令, wc [-lwc] file1 [file2 file3]
  • -l 只对行数进行统计 -w 只对单词数进行统计 -c 只对字符数进行统计。

  1. find命令 了解下即可

  2. sort命令 对文本内容进行排序，行排序。排序依据是ASCII码。

  3. grep命令 在一个或多个文件中搜索指定字符串，并显示包含字符串的行。

  • grep String file1 file2 …
  1. head命令 显示指定文件头部。
  2. tail命令 显示指定文件尾部。

输入输出重定向及管道

  1. 输出重定向

  • command >> filename 执行效果和 > 一样，不过是追加文件，而不是覆盖文件。

  • 进程的输入来自文件 command < filename
  1. 标准错误输出重定向 command 2> filename
  • 即将command错误运行时的错误日志写在filename中.
  1. 管道
  • command1 | command2 将command1的输出重定向作为command2的输入重定向。

shell可识别的命令形式

  1. 单条命令，即平常经常用的形式。

  2. 多条命令，可同时运行多条命令，提高键盘效率，需要注意的是，多条命令之间要以分号隔开.

  3. 复合命令,即管道前后的命令组合，形成更强大的命令。

  4. 后台命令,多线程兵法,数量不限，充分利用系统资源。

shell变量和引用符。

  • 系统的基本环境变量放在/etc/profile中，用户的环境变量放在用户主目录下的.profile。用户shell启动时，先执行系统环境变量，再执行用户环境变量。

  • 常用系统变量

  1. $0 当前shell程序的名称。
  2. $1 ~ $9 命令行上的第一个到第九个参数。
  3. $# 命令行上的参数个数。
  4. $* 命令行上的所有参数。

  • 局部变量

  • 用户可自定义的变量，定义的时候注意变量名和等号不能有空格，在shell编程里，空格就是分隔符号。

  • 引用局部变量的时候，一定要带上$符号。

  • 注意:单引号内部不能进行变量替换，而双引号则可以进行变量替换。

  • time命令: 显示当前shell命令花费的时间。

  • 值得注意的是 real > user + sys 因为real中包含了进程切换的等待时间。

进程监控

  • “进程”是可运行程序在内存中的一次运行实例。

  • 进程运行时都要经过类似: 创建 -> 运行 -> 等待 -> 运行 -> … -> 等待 -> 运行 -> 结束 的声明周期。

  • 于是周期大致可分为三种状态: 运行态,就绪态,睡眠态

  • ps命令:获取进程状态信息

  • 不带ps的命令显示该用户当前活动进程的基本信息.

  • -e (或-a) 显示系统中所有活动进程信息

  • -f 显示进程的所有信息.

  • sleep命令

  • sleep time -> 使进程暂停time秒

  • kill命令

  • 终止进程。

  • nice命令

  • 调整进程优先。
