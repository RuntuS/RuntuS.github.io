Nodejs模块化与文件读写

模块化

  • 在Js中也有模块化的运用，一般来说,一个文件就是一个模块。

  • 通过在一个文件中定义一系列的函数或变量，然后从其他文件引用，从而实现模块化。

模块化的条件

  1. 封装的文件里需要有export来暴露出本文件中的函数和变量。

  2. 引用的文件需要使用require来引用该模块文件从而获得暴露出去的函数和变量。

文件系统(fs : File System)

路径问题(这个很重要)

  • 在node里面要分清楚工作路径和文件路径，两者是不同的。
  • 程序内如果涉及到文件读写等一系列需要引用外部路径的操作时，其基准路径都是node的工作路径。
  • 所谓的工作路径是执行node命令时所在的路径。
  • $\color{red}{需要注意的是require的路径选择有所不同，require是以文件路径为基准的，与工作路径无关。}$
  • 详情可以查看浅谈NodeJs的几种文件路径

文件的读写(同步与异步)

同步读写

  • 读 : readFileSync , 写 : writeFileSync

异步读写

  • 读 : readFile ; 写 : writeFile

  • 需要注意的是，writeFile不等待回调是不安全的，比如上述的例子中，Data的值虽然打印出来确实是读取的值，但是真正写入文件的Data值是空的。原因是writeFile在写文件的时候，没有等待readFile执行完就开始执行，导致Data没有被成功赋值，所以是空的。
