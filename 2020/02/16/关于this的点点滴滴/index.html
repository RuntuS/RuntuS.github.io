关于this的点点滴滴

关于this

  • this在js里是个诡异的结构，它不是静态确定的，而是动态绑定对象。这就使得this的操作非常灵活。

    this默认绑定

  • 在非严格情况下，独立函数调用的情况下，this会绑定到全局变量上(浏览器 -> window , Node -> global)。

  • 但是如果是严格环境下，则this绑定的为undefined。

  • Node环境下js是模块化的，每一个Node文件就是一个模块，在一个js文件里用var定义的变量只是绑定到这个模块上，但是并不是绑定到global上，global还在这一个个模块之上。下面的例子将会更好理解。

  • 如果在严格环境下，this在默认情况下不会绑定到全局变量，而一直遵循默认绑定,如果外层是全局变量，则直接为undefined。

隐式绑定

  • 如果调用this的位置具有上下文，或者说被某个对象拥有或者包含，则this会绑定到这个上下文。

  • 此时的this会绑定到obj上。

隐式丢失

  • 但又是this绑定会丢失，原因是this绑定是动态的，它不同于Java中的this，绑定是谁就是谁。js中的this会随着调用环境的不同而指向不同的对象。

  • 如果要分析隐式绑定，一定要看它的函数引用以及执行的环境在哪里，进而推断出this的绑定情况。

new绑定

  • 通过new来调用函数，或者说发生构造函数调用时，会自动执行下面的操作。
  1. 创建一个全新的对象。
  2. 新对象会绑定到函数调用的this上。
  3. 如果这个函数没有返回其他对象，那么new表达式中的函数调用会自动返回这个新对象。

上面四种this绑定具有优先级分别是 new > 显式绑定 > 隐式绑定 > 默认绑定。

  • 具体可以参照js-1 P91-P93

补充: 何为ployfill？

  • ployfill代码主要用于旧浏览器的兼容，有些旧浏览器不兼容新的ES语法，则可以通过对应的ployfill代码去实现对应的新功能。
  • 比如说在不支持ES6语法的浏览器里手撸ployfill代码实现Promise(面试题的常问题)。
