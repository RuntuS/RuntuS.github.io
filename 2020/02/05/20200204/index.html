连接查询

内连接

  • 内连接的结果必须是两个表都同时具有的数据才行，即不能出现NULL。
  • 例如对students表和classes表两个表进行内连接，如果说students表中class_id是1，2，3，而classes的id有1，2，3，4。假如连接依据是 classes.id = students.class_id, 则classes中id=4的记录就不会被连接进去。

外连接

  • 外连接里又分左外连接，右外连接，全外连接。
  • 外连接即允许NULL的存在，如果一个表(假设为表1)中的某记录在另一个表(假设为表2)中没有对应的记录匹配，则在连接的结果中，会把表1中的无法在表2里匹配的记录数据写入总表中，且其他有关表2的属性赋值为NULL。

    左外连接是选出左表(相当于上述表1)中的特殊记录。

    右外连接是选出右表(相当于上述表1)中的特殊记录。

    全外连接就是两表都同时考虑。

SQL连接语法

  • 下面谈论内连接语法，其他连接语法与内连接基本相同(除了关键字不同)
  • 关键字 INNER JOIN -> 指定另一个连接表。
  • 关键字 ON -> 指定连接条件(注意和where查询条件区分开来，两者虽然都是条件，但是作用的地方不同。)
  1. 先确定主表，仍然使用FROM <表1>的语法；
  2. 再确定需要连接的表，使用INNER JOIN <表2>的语法；
  3. 然后确定连接条件，使用ON <条件…>，这里的条件是s.class_id = c.id，表示students表的class_id列与classes表的id列相同的行需要连接；
  4. 可选：加上WHERE子句、ORDER BY等子句。
  • 其他连接的关键字为右外连接:RIGHT OUTER JOIN, 左外连接: LEFT OUTER JOIN, 全外连接: FULL OUTER JOIN
