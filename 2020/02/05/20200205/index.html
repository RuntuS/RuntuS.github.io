跨域问题 + 四种npm安装区别 + express路由

跨域问题问题解决
原文章链接

npm install moduleName 命令

  1. 安装模块到项目node_modules目录下。
  2. 不会将模块依赖写入devDependencies或dependencies 节点。
  3. 运行 npm install 初始化项目时不会下载模块。

npm install -g moduleName 命令

  1. 安装模块到全局，不会在项目node_modules目录中保存模块包。
  2. 不会将模块依赖写入devDependencies或dependencies 节点。
  3. 运行 npm install 初始化项目时不会下载模块。

npm install -save moduleName 命令

  1. 安装模块到项目node_modules目录下。
  2. 会将模块依赖写入dependencies 节点。
  3. 运行 npm install 初始化项目时，会将模块下载到项目目录下。
  4. 运行npm install –production或者注明NODE_ENV变量值为production时，会自动下载模块到node_modules目录中。

npm install -save-dev moduleName 命令

  1. 安装模块到项目node_modules目录下。
  2. 会将模块依赖写入devDependencies 节点。
  3. 运行 npm install 初始化项目时，会将模块下载到项目目录下。
  4. 运行npm install –production或者注明NODE_ENV变量值为production时，不会自动下载模块到node_modules目录中。
  • 如果只是在开发环境下使用的插件，则就使用该安装模式。

总结

  • devDependencies 节点下的模块是我们在开发时需要用的，比如项目中使用gulp ，压缩css、js的模块。这些模块在我们的项目部署后是不需要的，所以我们可以使用 -save-dev 的形式安装。像 express 这些模块是项目运行必备的，应该安装在 dependencies 节点下，所以我们应该使用 -save 的形式安装。

Express 路由

正则表达式

  • 路由的路径可以是正则表达式，但是正则表达式的路径不再用引号扩起来，而是用斜线扩起来。

中间件

  • 路由函数express.Router()本身也是一个中间件函数，中间件存在于用法发出请求和最后响应发出这两个过程之间。

  • 其主要作用是形成一条中间件处理链，对请求信息做一系列的处理加工，最后再发送响应。

  • 正如备注所说，中间件处理链顺序是根据声明先后而确定的。并且通过next()传递，假如这里无next，则永远无法到达get请求，即会一直处于请求阶段。

  • 不仅仅考虑中间件的顺序，路由请求的顺序也要考虑进去，如果请求在整个中间件的前面，那么中间件会失效。(因为res.send已经把请求给结束了，请求响应无法传递下去。)

路由参数

  • 路径参数是命名的 URL 段，用于捕获在 URL 中的位置指定的值。命名段以冒号为前缀，然后是名称（例如 /:your_parameter_name/。捕获的值保存在 req.params 对象中，键即参数名（例如 req.params.your_parameter_name）。
