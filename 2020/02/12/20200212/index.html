map/reduce (注意和Map数据结构相区别开来)

  • map和reduce都是定义在Array里的方法，作用是对数组的每一个元素进行操作。

  • map倾向于对于每个元素的单独操作；reduce则倾向于对所有元素的整体进行操作。

  • map的回调函数只传入一个参数，参数对应的为数组中的每一个元素。

  • map返回一个全新的数组，与原数组没有任何关系。而全新数组的每个元素的值由回调函数中的return决定。

  • 这一点要格外注意，要与forEach()方法进行区别，forEach是直接在原数组上进行改变，不返回新数组。所以说，如果只是想在原数组上进行操作，那么就用forEach方法。

  • reduce的回调函数要传入两个参数，它是一种递归定义，第一个参数是前两个元素作用的结果(如果是数组开始元素，则前两个元素默认结果为0),第二个参数就是紧接着的下一个元素。

  • 练习题

  • 请把用户输入的不规范的英文名字，变为首字母大写，其他小写的规范名字。输入：[‘adam’, ‘LISA’, ‘barT’]，输出：[‘Adam’, ‘Lisa’, ‘Bart’]。

  • 这里有个常犯的错误，如果为了图方便，直接在原字符串上进行操作是不行的(字符串的每个元素是不能改变的可以查看上一篇有关于字符串的博客)。所以才考虑重新设定变量来存储中间值。

  • 补充两个方法。

  • 字符转换成十进制ASCII码 : ‘A’.charCodeAt() => 65

  • 十进制ASCII码转换成字符 : String.fromCharCode(65) // A

过滤器 filter

  • filter也是对每个元素进行操作。只不过它只是起一个筛选作用，满足条件的保留，不满足条件的就被剔除调。

sort Array的sort方法

  • 缺省的sort的排序是先把数组里的每个元素转化为字符串再进行排序。而字符串的排序是以ASCII码大小来进行排序的。
  • 所以缺省的sort一般不用于排序数字。
  • 但可以进行自定义的排序，向sort方法传入回调函数(处理函数).
